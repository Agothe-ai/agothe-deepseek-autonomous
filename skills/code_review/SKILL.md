# code_review
Review a Python file for common issues: syntax errors, unused imports, missing docstrings, long functions.

## When to use
When Paul says "review this code", "check my file", "any issues with X.py", or "code review".

## Tags
code, review, python, quality, lint, analysis

## Parameters (pass via context)
- context['file_path'] -- path to the Python file to review

## Output
Returns a structured review report.
